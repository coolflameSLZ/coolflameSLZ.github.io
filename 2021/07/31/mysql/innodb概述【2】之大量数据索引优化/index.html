数据量与B+树的层高关系

  1. 每个 page 都有一个 level，leaf page 的 level 是 0，root page 的 level 取决于整个 B+Tree 的高度。
  2. 因为页存储有 「空洞」 现象，所以不是非常固定的
  3. 根据主键的有序性，页分裂的情况，会少存许多数据。
  4. 一般来说 当数据为理论值的 2/3 时， 树就开始加一层了。

已知：

  • 单行数据为n byte ，每页存 (16000 / n ) 条记录，那么：
    • int 主键，每页1203个子节点索引指针。
    • bigint 主键，每页900个子节点索引指针。
    • 单行数据为 n byte ，每个page存 (16000 / n ) 条记录，
  • 假设1 row数据 300byte，每个page 存50条 数据。

计算公式为 ：

总行数 = （每页指针数） ^（几层） 每页行数*
