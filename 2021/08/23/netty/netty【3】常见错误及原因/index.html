netty【3】常见错误及原因

ChannelHandler收不到消息

常见原因：

  1. 服务端解码 ChannelHandler 存在缺陷，解码失败
  2. 客户端发送的消息畸形，导致服务端无法正确解码
  3. 服务端 NIO线程被阻塞，导致无法读取消息，业务线程池队列积压，导致消息排队或者被丢弃。
  4. 网络拦截

定位套路：

  1. 从第一个 ChannelHandler 开始 debug
  2. 再解码器，业务handler之前加入 loggingHandler
  3. 查看 NioEventLoop 线程状态
  4. 通过 tcpdump 抓包查看是否发送成功

内存泄漏

  1. 通过 jmap -dump:format=b,file=xx.pid 打印内存堆栈，使用 memoryAnalyzer 工具进行内存占用分析，查看泄漏点。

性能问题

  1. 使用火焰图定位cpu问题。
